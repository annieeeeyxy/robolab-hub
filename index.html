Intro · About the project

What is RoboPrompt?

An AI assistant for designing robotic arm control systems — from a single photo to architecture, code, and a test plan.

Try it ↗
The idea

Snap a photo of any robotic arm, answer only the questions the photo can't, and get a working control plan for it.

How to use RoboPrompt

  1. Prepare a clear photoFrame the full robotic arm so its joints, motors, controller, and end-effector are as visible as possible.
  2. Upload it to RoboPromptStart a new analysis and let the vision system identify the arm's brand, structure, joint count, and motor types.
  3. Review the hardware analysisConfirm what RoboPrompt detected and correct anything that does not match the machine in front of you.
  4. Fill in the missing detailsAnswer the focused questions about controllers, drivers, communication protocols, and the motion you want to achieve.
  5. Use and test the planFollow the generated architecture, code, wiring, calibration, and test steps; verify connections and begin at low speed.

The problem

Most robot projects get the hardware moving, then stall at the control system — firmware, drivers, and interfaces. That's because the mechanical structure, hardware information, communication protocol, and code are usually disconnected.

Generic AI tools lack robotics context: they guess code instead of asking what your machine actually is.

The approach

RoboPrompt refuses to guess. It identifies your hardware from the photo first — brand, joint count, motor type, structure — then asks for whatever is still missing before writing a single line of code.

The result is a plan that matches the machine in front of you: architecture, runnable code, wiring, and calibration.

Who it's for

Anyone with a robotic arm and no control system yet — from hobbyists with a servo kit on an Arduino or ESP32, to teams running brushless-motor industrial arms on Jetson or ROS2. Available in English, Español, Français, and 中文.

Intro · RoboLab & FIRST Tech Challenge

What is RoboLab for?

What this simulator does, and how the real FTC competition works — in plain words.

Try it ↗
In one sentence

RoboLab is a free virtual FTC field: write robot code, run it, and see your score — no real robot needed.

What you can do with it

  • Run Autonomous and TeleOp code on a 3D DECODE field
  • Pick a robot preset, alliance color, and start position
  • Watch live telemetry while your robot drives
  • See your score counted with real DECODE point values
  • Replay runs and get AI feedback to fix your code
  • Choose Sandbox (everything unlocked) or Learning Mode (step-by-step levels)

What is FTC?

FIRST Tech Challenge is a worldwide robotics competition for students. Teams build and program their own robot, then compete 2v2 — a Red and a Blue alliance, two teams each. Every season is a new game; this year's is DECODE™.

The DECODE game

Score purple and green artifacts in your goal, build the randomized 2-purple-1-green motif pattern on your ramp, and get both robots back to base before time runs out. Most points wins.

Match format · 2:30 total

30-second AUTO period → 8-second transition → 2-minute TELEOP period.

How a match plays

  1. AUTO (0:30)Robots run fully autonomously: use sensors to decode the motif, leave the launch line, and start scoring artifacts.
  2. TELEOP (2:00)Drivers take control. Robots collect and score artifacts; drive team members load artifacts from the loading zone.
  3. EndgameBuild the motif pattern on the ramp and return both robots to base — the alliance with the most points wins.

Scoring at a glance

Leave3 pts — robot off the launch line at the end of AUTO.
Classified3 pts per artifact sorted correctly through the goal.
Overflow1 pt per extra artifact through the goal.
Depot1 pt per artifact in the depot (TELEOP).
Pattern2 pts per artifact matching the motif on the ramp.
Base5 pts partial / 10 pts full return; +10 bonus if both robots fully return.
RankingExtra ranking points for winning and for hitting movement, goal, and pattern thresholds.

Rules summarized from the official DECODE Competition Manual — RoboLab scores with these same values.

Demo · Tutorial

How to use RoboPrompt

From a single photo to a control plan for your arm's end-effector — here's the full walkthrough.

Try it ↗
Guiding principle

“Never let the AI guess robot code directly. Identify the hardware first, then ask for whatever is still missing.”

Step by step

  1. Snap or find a photo of your armHobby kit, custom build, or industrial arm — any clear photo works.
  2. Upload itVision analysis identifies the brand, joint count, motor type, and mechanical structure.
  3. Answer only what's missingFindings are matched against known products; RoboPrompt asks just for the specs it couldn't infer.
  4. Get your control pathYour build is classified onto the right stack — Arduino, ESP32, ROS, or an industrial system.
  5. Receive the deliverablesA concrete architecture, runnable code, wiring instructions, and a calibration and test plan.

Why it exists

Getting hardware to move is the easy part — the control system is where projects stall. Robot programming is hard because the mechanical structure, hardware information, communication protocol, and code are usually disconnected. Generic AI tools lack that context and guess instead of asking.

Supported hardware

Category AHobby and custom builds — servo-driven arms on microcontrollers such as Arduino or ESP32.
Category BLarger industrial systems — brushless motors and reducers on platforms like Jetson or ROS2.

How to use RoboLab FTC

Configure a robot, run your code on the virtual DECODE field, and improve every run with data.

Try it ↗
The RoboLab loop

Define the goal → simulate the robot → learn from the data.

Step by step

  1. Pick your workspaceSandbox gives you every simulator control at once; Learning Mode unlocks features level by level.
  2. Set up the simulationChoose Autonomous or TeleOp, a robot preset (e.g. Turret Shooter — mecanum drive, continuous intake, indexed feeder, adjustable hood), your alliance color, and a start position (X, Y, heading).
  3. Write your robot codeCommands like driveToPosition(72, 90, 145); spinFlywheel(2400); shoot(60);
  4. Run it on the DECODE fieldYour program executes on the interactive 3D field while live telemetry streams in.
  5. Learn from the dataCheck the scoring tracker, review telemetry and playback, get AI feedback — then debug and run again.

Two ways in

Sandbox — the open workspace: all controls, both modes, telemetry, scoring, playback, and AI feedback with no restrictions.

Learning Mode — a guided progression for drivers and programmers who want a structured route from first drive command to full match strategy.

Simulator at a glance

Field3D DECODE field (2025–2026 season) with orbital camera controls.
ModesAutonomous and TeleOp.
ScoringGoals, total points, classified artifacts (3 pts each), overflow (1 pt each).
TelemetryLive robot data with timing and custom monitors.
